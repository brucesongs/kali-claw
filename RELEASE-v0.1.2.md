## 这一次，我们解决了什么问题？

v0.1.1 让 kali-claw 从一个攻击技术全面的工具库升级为一个从侦察到报告全流程覆盖的渗透测试智能体。但在实际使用中，我们发现了新的挑战：

> **能做，但还不够好、不够安全。**

具体来说：

- 发现了漏洞，但无法系统化地验证是否为真实漏洞还是扫描器误报
- 面对大量目标的批量任务，只能一个个手动操作，效率低下
- 每次渗透测试积累的经验无法沉淀，跨会话知识不断流失
- 想要练习或验证技术，但没有安全的本地靶场环境
- 自动化操作缺少安全护栏，存在越界风险

v0.1.2 正是为解决这五个问题而来。

## 五大新技能一览

### 1. Verification Loop — 多阶段验证协议

**一句话：每个发现都必须经过独立验证才能进入报告。**

六阶段验证流程：前置条件检查 → 执行观察 → 后置条件确认 → 独立方法复现 → 假阳性消除 → 证据文档。要求每个发现至少通过两种独立方法确认。

**典型场景：** 漏洞扫描器报告 10 个 SQL 注入点，验证循环逐个用不同方法独立确认，最终证实 3 个为真实漏洞，7 个为误报——避免在报告中提交无法重现的发现。

---

### 2. Autonomous Loops — 安全自主执行

**一句话：重复性任务自动化，但每一步都在控制之中。**

四种循环模式：顺序管道（逐目标执行）、监控循环（等待条件触发）、批处理（并行处理+限速）、学习循环（根据反馈迭代优化）。每种模式内建范围锁、速率限制、证据日志和错误处理。

**典型场景：** 对一个 C 段进行端口扫描，使用顺序管道模式逐主机扫描，每步自动记录证据，遇到目标异常立即停止并报告。

---

### 3. Continuous Learning — 持续学习

**一句话：把每次渗透测试的经验变成可复用的知识。**

学习循环：模式检测 → 结构化提取 → 置信度评分 → 分层存储 → 交叉引用。三级记忆层：短期战术记忆（当前任务）、中期技术记忆（跨任务可复用）、长期战略记忆（安全趋势洞察）。

**典型场景：** 在多次测试中发现某 WAF 对特定绕过技术的拦截规律，系统自动提取此模式并标记为高置信度知识，下次遇到相同 WAF 时直接推荐有效绕过方法。

---

### 4. Docker Patterns — 安全测试实验室

**一句话：一键搭建本地靶场，安全验证所有攻击技术。**

五种 Docker 模式：脆弱 Web 应用靶场（DVWA、SQLi-Labs、Juice Shop）、网络渗透靶场、多阶段攻击链靶场、一次性测试、工具校准环境。所有靶场仅绑定 127.0.0.1，确保绝不暴露到公网。

**典型场景：** 想要练习 SQL 注入，一条命令启动 DVWA 靶场，在完全隔离的环境中安全实践，练完一条命令销毁环境。

---

### 5. Safety Guard — 安全护栏

**一句话：为所有操作设置安全边界，防止越界和意外。**

三级安全模式：谨慎模式（常规检查）→ 冻结模式（需确认才继续）→ 守卫模式（直接拦截）。危险命令拦截表、参与规则模板、三级事件响应协议（轻微/服务影响/严重事件）。

**典型场景：** 自动化批量扫描执行前，安全护栏自动检查每个目标是否在授权范围内，拦截超出范围的命令，检测到 IDS 触发时立即冻结所有操作。

## 五大能力维度

```
质量保障：verification-loop 确保发现可靠、报告可信
自动化：autonomous-loops 提供安全的批量执行框架
经验沉淀：continuous-learning 将操作经验转化为结构化知识
训练环境：docker-patterns 提供标准化的隔离靶场
安全边界：safety-guard 为所有操作设置护栏
```

## 技能协同示例

### 漏洞验证流程

```
vulnerability-assessment（自动化扫描）
  → verification-loop（独立确认 + 假阳性消除）
    → security-bounty-hunter（筛选 + PoC + 提交报告）
```

### 自动化批量扫描

```
safety-guard（范围检查 + 安全前置）
  → autonomous-loops（安全批量执行）
    → terminal-ops（证据链记录）
```

### 经验积累闭环

```
web-sqli / web-xss（攻击技术实践）
  → verification-loop（验证发现）
    → continuous-learning（提取模式 → 存储为可复用知识）
```

### 本地靶场训练

```
docker-patterns（启动 DVWA 靶场）
  → web-sqli（在隔离环境练习注入）
    → verification-loop（验证注入效果）
```

## 未来路线图

v0.1.2 同步发布了 kali-claw 的技能演进路线图，9 个未来技能按优先级分三个梯队：

**第二梯队（中优先级）：** `article-writing`（安全报告撰写）、`browser-qa`（浏览器安全测试）、`data-scraper-agent`（结构化数据提取）、`exa-search`（高级网络搜索）

**第三梯队（未来探索）：** `mcp-server-patterns`（MCP 服务器集成）、`council`（多视角分析）、`strategic-compact`（战略上下文管理）
