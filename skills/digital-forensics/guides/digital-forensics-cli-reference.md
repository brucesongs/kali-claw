# Digital Forensics Command Line Tools Detailed Reference

## 1. SleuthKit - Filesystem Forensics Analysis Toolset

### Core Tool Overview
SleuthKit provides a complete set of filesystem analysis tools, supporting multiple filesystem formats (FAT, NTFS, Ext2/3/4, HFS+, APFS, etc.).

### Main Tools and Usage

#### mmls - Display Partition Table Information
```bash
# Display disk image partition layout
mmls disk_image.dd

# Specify offset
mmls -o 63 disk_image.dd

# Detailed output
mmls -v disk_image.dd
```

#### fsstat - Display Filesystem Statistics
```bash
# Analyze filesystem structure
fsstat disk_image.dd

# Specify filesystem type and offset
fsstat -f ntfs -o 32256 disk_image.dd

# Detailed analysis
fsstat -v disk_image.dd
```

#### fls - List Filesystem Contents
```bash
# List root directory contents
fls disk_image.dd

# Recursively list all files
fls -r disk_image.dd

# Only display deleted files
fls -d disk_image.dd

# Detailed listing (similar to ls -l)
fls -l disk_image.dd

# MAC timeline format output
fls -m /mnt/point disk_image.dd
```

#### icat - Extract File Contents
```bash
# Extract file content by specified inode
icat disk_image.dd 12345 > recovered_file.txt

# Specify offset
icat -o 32256 disk_image.dd 12345

# Extract to standard output
icat disk_image.dd 12345 | strings
```

#### ifind - Find Inodes
```bash
# Find inode by filename
ifind -n "filename.txt" disk_image.dd

# Find inode by data block
ifind -d 1000 disk_image.dd
```

#### ils - List Inode Information
```bash
# List all inodes
ils disk_image.dd

# Detailed inode information
ils -l disk_image.dd
```

### Practical Workflow
```bash
# Step 1: Analyze disk layout
mmls evidence.dd

# Step 2: Determine filesystem offset
# Assume NTFS partition starts at sector 63, each sector 512 bytes
OFFSET=$((63 * 512))

# Step 3: Analyze filesystem
fsstat -o $OFFSET evidence.dd

# Step 4: List file contents
fls -o $OFFSET -r evidence.dd

# Step 5: Extract specific file
icat -o $OFFSET evidence.dd <inode_number> > recovered_file
```

## 2. Autopsy - Digital Forensics Platform (Command Line Mode)

### Basic Usage
```bash
# Start Autopsy web interface
autopsy

# Specify port
autopsy -p 8080

# Specify evidence storage directory
autopsy -d /path/to/evidence_locker

# Remote access
autopsy remote_host
```

### Command Line Options
- `-c`: Force inclusion of cookies in URLs
- `-C`: Force exclusion of cookies in URLs
- `-d dir`: Specify evidence storage directory
- `-p port`: Specify server port (default 9999)
- `remoteaddr`: Specify browser host (default localhost)

### Practical Points
- Autopsy primarily operates through its web interface
- Command line is mainly used for starting services and configuration
- Requires coordination with SleuthKit toolset
- Supports multi-user collaboration and case management

## 3. Scalpel - File Carving Tool

### Basic Usage
```bash
# Carve using default configuration file
scalpel disk_image.dd

# Specify custom configuration file
scalpel -c custom.conf disk_image.dd

# Specify output directory
scalpel -o /recovered_files disk_image.dd

# Preview mode (no actual carving)
scalpel -p disk_image.dd
```

## 4. Bulk Extractor - High-Efficiency Digital Forensics Tool

### Basic Usage
```bash
# Basic scan
bulk_extractor image.dd

# Enable specific scanners
bulk_extractor -e email -e credit_card image.dd

# Disable specific scanners
bulk_extractor -x exif image.dd

# Specify output directory
bulk_extractor -o /output/dir image.dd

# Search for specific pattern
bulk_extractor -f "pattern" image.dd
```

### Scanner Management
```bash
# List all available scanners
bulk_extractor -L

# Only enable specific scanners
bulk_extractor -E email image.dd

# Disable all scanners, then enable specific ones
bulk_extractor -x all -e url image.dd
```

### Advanced Features
- **Context window**: `-C 32` sets 32-byte context window
- **Page handling**: `-G 16M` sets 16MB page size
- **Carving mode**: Supports carving for each file format
- **Multi-threading**: Automatically utilizes multi-core processors

### Output Files
Bulk Extractor generates multiple output files:
- `email.txt`: Extracted email addresses
- `url.txt`: Extracted URLs
- `credit_card.txt`: Credit card numbers
- `gps.txt`: GPS coordinates
- `exif.txt`: EXIF metadata

## 5. ExifTool - Metadata Analysis Tool

### Basic Usage
```bash
# Display all metadata
exiftool image.jpg

# Only display specific tags
exiftool -Make -Model -DateTimeOriginal image.jpg

# Batch processing
exiftool *.jpg

# Recursively process directory
exiftool -r /path/to/photos/
```

### Advanced Features
```bash
# Modify metadata
exiftool -Artist="Photographer Name" image.jpg

# Delete all metadata
exiftool -all= image.jpg

# Copy metadata
exiftool -tagsFromFile source.jpg destination.jpg

# Export as JSON
exiftool -json image.jpg
```

### Supported Formats
ExifTool supports over 200 file formats, including:
- Image formats: JPEG, PNG, TIFF, RAW, etc.
- Video formats: MP4, AVI, MOV, etc.
- Document formats: PDF, DOC, XLS, etc.
- Audio formats: MP3, WAV, FLAC, etc.

## 6. PhotoRec - Data Recovery Tool

### Basic Usage
```bash
# Interactive recovery
photorec disk_image.dd

# Specify recovery directory
photorec -d /recovered/ disk_image.dd

# Enable logging
photorec /log disk_image.dd
```

### File Type Support
PhotoRec supports recovering over 480 file formats, including:
- Documents: DOC, PDF, XLS, PPT, etc.
- Images: JPEG, PNG, GIF, TIFF, etc.
- Audio: MP3, WAV, FLAC, etc.
- Video: MP4, AVI, MOV, etc.
- Archives: ZIP, RAR, 7Z, etc.

### Recovery Strategy
- **File signature based**: Identifies file types through header/footer signatures
- **Ignore filesystem**: Directly scans disk data
- **Smart recovery**: Handles fragmented files

## 7. TestDisk - Partition Recovery Tool

### Basic Usage
```bash
# Interactive partition recovery
testdisk disk_image.dd

# List current partitions
testdisk /list disk_image.dd

# Enable logging
testdisk /log disk_image.dd
```

### Features
- **Partition table repair**: Fix corrupted partition tables
- **Lost partition recovery**: Find deleted partitions
- **Boot sector repair**: Fix boot issues
- **Filesystem support**: FAT, NTFS, Ext2/3/4, HFS+, etc.

### Workflow
1. **Analyze**: Scan disk to find partitions
2. **Select**: Select partition to recover
3. **Write**: Write partition table back to disk
4. **Verify**: Verify recovery results

## Tool Combination Usage Strategy

### 1. Complete Forensics Workflow
```bash
# Step 1: Disk image analysis
mmls evidence.dd
fsstat -o OFFSET evidence.dd

# Step 2: Filesystem analysis
fls -o OFFSET -r evidence.dd > file_list.txt
strings evidence.dd > strings_output.txt

# Step 3: File carving
scalpel -o carved_files evidence.dd
photorec -d recovered_files evidence.dd

# Step 4: Metadata analysis
exiftool recovered_files/*.jpg > metadata.txt

# Step 5: Specific data extraction
bulk_extractor -o bulk_results evidence.dd
```

### 2. Rapid Response Workflow
```bash
# Emergency data recovery
photorec -d /urgent_recovery/ damaged_disk.dd

# Critical information extraction
bulk_extractor -e email -e url -e credit_card evidence.dd

# Partition fix
testdisk /list damaged_disk.dd
```

### 3. Deep Analysis Workflow
```bash
# Detailed filesystem analysis
for inode in $(fls -d evidence.dd | awk '{print $2}' | cut -d: -f1); do
    icat evidence.dd $inode > deleted_file_$inode

exiftool -csv recovered_files/* > timeline.csv

# Network activity reconstruction
bulk_extractor -e url -e email -e domain evidence.dd
```

## Best Practices and Considerations

### 1. Legal and Ethics
- Always Obtain Legal Authorization
- Maintain Chain of Custody
- Handle Original Evidence in Read-Only Mode
- Document All Operational Steps

### 2. Technical Best Practices
- Create Hash of Original Evidence
- Use Write-Protected Devices
- Analyze on Copies
- Verify Accuracy of Tool Output

### 3. Performance Optimization
- Select Appropriate Tools Based on Evidence Size
- Use Multi-threaded Tools to Improve Efficiency
- Reasonably Allocate System Resources
- Monitor Disk Space Usage

### 4. Result Validation
- Cross-validate Results from Different Tools
- Manually Verify Key Findings
- Maintain Detailed Analysis Logs
- Document Analysis Hypotheses and Conclusions
